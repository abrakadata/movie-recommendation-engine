# Movie Buddy App

## Overview

The refactor of the existing codebase for the local Streamlit app that recommends movies based on movies the user likes, consists of the following:
We will use the MPST dataset instead of the tmdb_5000_movies.csv, will continue to use sentence-transformer embeddings and cosine similarity but now in TWO Discovery Modes:
1. Free-text query and 
2. Movie-to-movie similarity search.

---

## Dataset

- **File:** `mpst_full_data.csv` available in project root
- **Rows:** 14,828 movies
- **Columns used:**
  - `imdb_id` — unique identifier
  - `title` — movie name
  - `plot_synopsis` — full plot summary (primary embedding source)
  - `tags` — comma-separated semantic labels (e.g. `murder, romance, flashback`)
  - `split` — train / test / val (used for display info only, not filtered)
  - `synopsis_source` — imdb or wikipedia

## Step 1 — Precompute Embeddings (`precompute.py`)

Run once before launching the app. Encodes all 14,828 plot synopses and saves
results to disk.

### Responsibilities

1. Load `mpst_full_data.csv` with pandas
2. Drop rows where `plot_synopsis` or `title` is null
3. Deduplicate on `imdb_id` (keep first)
4. Strip whitespace from `title` and `tags` columns
5. Load model `all-MiniLM-L6-v2` from `sentence-transformers`
6. Encode `plot_synopsis` column in batches of 64, with `show_progress_bar=True`. On the Streamlit UI, the progress bar WILL NOW PERCENTAGE COMPLETE.
7. Save embeddings as `embeddings.npy` (float32, shape: [N, 384])
8. Save cleaned dataframe as `movies.pkl`
9. Print confirmation on the screen: number of movies encoded, time elapsed.


Expected output right below the progress bar, at its beginning:

```
Loading dataset...
Encoding 14,828 synopses with all-MiniLM-L6-v2...
Saved embeddings.npy (21.8 MB)
Saved movies.pkl (18.4 MB)
Done.

```

Expected output at the END of progress bar:
```
100%|████████████████| 232/232 35.2 sec
```

## App Architecture

### Entry Point (`app.py`)

`app.py` contains only the navigation definition:

```python
pages = st.navigation([
       st.Page("pages/free-search-page.py", title="Free-text search", icon="🔍"),
       st.Page("pages/similarity-page.py",  title="Movie similarity", icon="🎬"),
])
pages.run()
```

No logic, no data loading lives here.

### Shared Startup (cached, runs once per session)

Both pages import from a shared `utils.py` module (or inline with caching):

```
load_model()     →  SentenceTransformer("all-MiniLM-L6-v2")   @st.cache_resource
load_data()      →  df from movies.pkl + embeddings from embeddings.npy   @st.cache_data
```

At runtime, the app reads only `movies.pkl` and `embeddings.npy`. The two CSV files remain raw inputs for `precompute.py`; no extra intermediate joined CSV is created.

If `embeddings.npy` or `movies.pkl` are missing, run `python precompute.py` first.

## Page 1 — Free-Text Search (`pages/free-search-page.py`)

### UI Elements

- `st.text_area` labeled "Describe the kind of movie you want to watch"
  - Placeholder: `"A psychological thriller where nothing is what it seems"`
  - Height: 100px
- `st.button("Find Movies")` — triggers search

### Logic

```
1. Validate: query string must be non-empty (show st.warning if blank)
2. Encode query:
       query_vec = model.encode([query_text])   # shape (1, 384)
3. Compute cosine similarity:
       content_scores = cosine_similarity(query_vec, embeddings)[0]  # shape (N,)
4. Compute hybrid scores:
       hybrid = similarity_weight * content_scores + pop_weight * df["pop_score"].values
5. Apply tag filter (if tags selected):
       keep rows where ANY selected tag appears in movie's tags string
       zero out hybrid scores for excluded rows
6. Apply min similarity threshold (zero out rows where content_scores < threshold)
7. Rank: argsort descending, take top-N
8. Display results (see Results Table spec below)
```

### Edge Cases

- Empty query → `st.warning("Please enter a description.")`
- No results after filtering → `st.info("No movies matched your filters. Try relaxing the tag filter or lowering the similarity threshold.")`

## Page 2 — Movie Similarity (`pages/similarity-page.py`)

### UI Elements

- `st.selectbox` labeled "Pick a movie you like"
  - Options: sorted list of all movie titles from `df["title"]`
  - Searchable (Streamlit selectbox is searchable by default)
- Search triggers automatically on selection change (no button needed)

### Logic

```
1. Look up selected title's index in df
2. Retrieve its pre-computed embedding:
       seed_vec = embeddings[idx].reshape(1, -1)   # shape (1, 384)
3. Compute cosine similarity against all embeddings
4. Compute hybrid scores:
       hybrid = similarity_weight * content_scores + pop_weight * df["pop_score"].values
5. Zero out hybrid score for the selected movie itself (don't recommend itself)
6. Apply tag filter (same as Page 1)
7. Apply min similarity threshold (zero out rows where content_scores < threshold)
8. Rank: argsort descending, take top-N
9. Show the seed movie's details in a highlighted info box above results
10. Display results (see Results Table spec below)
```

### Seed Movie Info Box

Display above results using `st.info()` or a styled `st.container()`:

```
🎬  <Title>

<First 200 characters of synopsis>...
```

## Results Table (shared by both modes)

Results rendered as a single `st.dataframe` with `use_container_width=True` and `hide_index=True`.

### Columns

| Column | Source field | Type | Notes |
|---|---|---|---|
| Title | `title` | TextColumn | medium width |
| Tags | `tags` | TextColumn | comma-separated; medium width |
| Match | computed cosine similarity | TextColumn | formatted as percentage + color emoji prefix (see below); small width |
| Synopsis | `plot_synopsis` | TextColumn | first 200 characters + "…"; large width |

### Similarity Score Formatting

The `Match` column value is a plain string combining a color indicator and the percentage:

| Score | Formatted value |
|---|---|
| ≥ 0.75 | `🟢 87%` |
| 0.50 – 0.74 | `🟡 63%` |
| < 0.50 | `🔴 41%` |

The percentage is `round(score * 100)`. No `ProgressColumn` — the emoji prefix carries the signal so the column stays narrow and readable.

## Hybrid Scoring

### Popularity Data Source

The MPST dataset has no vote data. Popularity scores are obtained by a **title-based join** with `movies_metadata.csv` (the existing TMDB dataset) during `precompute.py`:

```
1. Load movies_metadata.csv, keep title / vote_average / vote_count
2. Normalize titles (lowercase, strip) in both datasets
3. Left-join MPST df on normalized title
4. For unmatched rows: fill vote_average with median, vote_count with 0
5. Compute pop_score = (vote_average / 10) * log1p(vote_count) / max(log1p(vote_count))
6. Store the cleaned MPST rows plus pop_score in movies.pkl
```

Unmatched movies (no TMDB entry found) get `pop_score = 0.0`, so they rank purely on content similarity. `movies_metadata.csv` stays in the repo as a raw input; `movies.pkl` is the final joined runtime artifact.

### Blending

```python
hybrid = similarity_weight * content_scores + (1 - similarity_weight) * df["pop_score"].values
```

`similarity_weight` comes from the sidebar slider (default 0.75). Blending happens at query time so the user can adjust the slider without invalidating the embedding cache.

## Tag Filter Behavior

- Tag filter lives in the sidebar and applies to both modes
- If no tags selected → no filtering (return all results)
- If tags selected → a movie must contain **at least one** selected tag to appear
- Tag matching: case-insensitive substring check against the raw `tags` string

## Performance Notes

- Embeddings fit in RAM: ~21 MB for 14,828 × 384 float32 values
- Cosine similarity over 14,828 vectors is ~2ms with sklearn on CPU
- Model load time (~1–2s) is hidden behind `st.cache_resource`
- No GPU required; runs on any modern laptop

## Error States

| Condition | Behavior |
|---|---|
| `embeddings.npy` missing | `st.error("Run python precompute.py first.")` + `st.stop()` |
| `movies.pkl` missing | Same as above |
| `mpst_full_data.csv` missing (precompute) | Print clear message and exit with code 1 |
| Query too short (< 3 chars) | `st.warning("Query too short.")` |
| Model download fails (no internet) | Caught exception → `st.error("Could not load model. Check internet connection.")` |
